## 1. 需求总结

| 维度 | 选择 |
|------|------|
| **应用类型** | Web + 桌面端（同一套代码，Tauri 打包桌面） |
| **数据存储** | 本地优先，支持统计与历史 |
| **核心功能** | 番茄计时 + 白噪音 + 任务管理 |
| **技术栈** | Vue 生态 |

### 功能清单

- ⏱️ **番茄计时器**：25 分钟专注 + 5 分钟短休息 + 15 分钟长休息（可自定义）
- ✅ **任务管理**：创建任务、关联番茄、追踪进度
- 📊 **数据统计**：专注时长趋势、打卡日历、标签分布
- 🎵 **白噪音**：多种环境音、支持混合播放、音量控制
- 🖥️ **桌面端**：系统托盘、原生通知、全局快捷键

## 2. 技术选型

### 2.1 前端框架

| 层 | 选择 | 理由 |
|----|------|------|
| **框架** | Vue 3 + Composition API | 响应式 API 适合计时器状态管理 |
| **构建工具** | Vite | Vue 官方推荐，HMR 极快，构建产物小 |
| **语言** | TypeScript | 类型安全，IDE 补全好，长期维护成本低 |
| **UI 组件库** | Naive UI | Vue 3 原生、Tree-shaking 友好、主题定制灵活、体积小 |
| **CSS 方案** | UnoCSS（或 Tailwind CSS） | 原子化 CSS，配合 Naive UI 做微调样式 |
| **路由** | Vue Router 4 | 页面切换（计时器 / 统计 / 设置 / 任务） |
| **状态管理** | Pinia | Vue 3 官方推荐，TS 支持好，轻量 |

### 2.2 桌面端封装

| 选择 | 理由 |
|------|------|
| **Tauri v2** | Rust 后端，打包体积极小（~3-5MB vs Electron ~150MB），系统资源占用低，原生通知能力强，支持系统托盘 |

> **备选方案**：如果更熟悉 JS 全栈，Electron 也可以，但体积和性能都不如 Tauri。

### 2.3 数据存储

| 方案 | 用途 |
|------|------|
| **IndexedDB**（via Dexie.js） | 浏览器端存储番茄记录、任务、设置，支持离线 |
| **SQLite**（via Tauri Plugin） | 桌面端持久化，更可靠 |

> **策略**：抽象一个 `StorageAdapter` 接口，Web 端用 Dexie/IndexedDB，桌面端用 SQLite，上层代码无感知。

### 2.4 音频方案

| 方案 | 说明 |
|------|------|
| **Web Audio API** + 内置音频资源 | 白噪音（雨声、海浪等）使用预置 MP3/OGG 文件，通过 Web Audio API 播放和混合 |
| **Howler.js**（可选） | 跨浏览器音频库，简化音量控制和淡入淡出 |

### 2.5 其他工具

| 工具 | 用途 |
|------|------|
| **Vitest** | 单元测试 |
| **Playwright** | E2E 测试（Web + 桌面） |
| **unplugin-auto-import** | 自动导入 Vue/Pinia API，减少样板代码 |
| **vue-i18n**（可选） | 国际化支持 |

## 4. 核心模块设计

### 4.1 计时器模块 (`useTimer`)

#### 状态机

```
                    ┌─────────────────────────────────────┐
                    │                                     │
                    ▼                                     │
    ┌──────┐  start()  ┌─────────┐  pause()  ┌────────┐  │
    │ IDLE │ ────────→ │ RUNNING │ ────────→ │ PAUSED │  │
    └──────┘          └────┬────┘          └───┬────┘    │
                           │ time up           │         │
                           ▼                   │ resume()│
                    ┌────────────┐              │         │
                    │ BREAK_IDLE │              │         │
                    └──────┬─────┘              │         │
                     start()│                   │         │
                           ▼                   │         │
                    ┌──────────────┐  pause()  │         │
                    │ BREAK_RUNNING│ ──────→ BREAK_PAUSED ┘
                    └──────┬──────┘    resume()
                    time up│
                           ▼
                    ┌──────────┐
                    │ → IDLE   │  (自动进入下一个番茄)
                    └──────────┘
```

#### 状态定义

```typescript
type TimerStatus =
  | 'idle'           // 等待开始
  | 'running'        // 专注中
  | 'paused'         // 专注暂停
  | 'break_idle'     // 休息等待
  | 'break_running'  // 休息中
  | 'break_paused'   // 休息暂停

interface TimerState {
  status: TimerStatus
  remaining: number          // 剩余秒数
  totalDuration: number      // 当前阶段总时长（秒）
  currentPomodoro: number    // 当前是第几个番茄（从 1 开始）
  pomodoroCount: number      // 一轮番茄总数（默认 4）
}

interface TimerConfig {
  focusDuration: number      // 专注时长，默认 25 * 60
  shortBreakDuration: number // 短休息，默认 5 * 60
  longBreakDuration: number  // 长休息，默认 15 * 60
  longBreakInterval: number  // 每几个番茄一次长休息，默认 4
  autoStartBreak: boolean    // 休息是否自动开始，默认 false
  autoStartPomodoro: boolean // 番茄是否自动开始，默认 false
}
```

#### 核心方法

```typescript
interface UseTimerReturn {
  // 状态（响应式）
  status: Ref<TimerStatus>
  remaining: Ref<number>
  progress: ComputedRef<number>       // 0 ~ 1 的进度百分比
  formattedTime: ComputedRef<string>  // "25:00" 格式

  // 操作
  start: () => void
  pause: () => void
  resume: () => void
  reset: () => void
  skip: () => void                    // 跳过当前阶段

  // 配置
  config: Ref<TimerConfig>
  updateConfig: (config: Partial<TimerConfig>) => void
}
```

#### 计时精度方案

```typescript
// 使用 Date.now() 差值校准，避免 setInterval 漂移
let lastTick = Date.now()

const tick = () => {
  const now = Date.now()
  const elapsed = Math.floor((now - lastTick) / 1000)
  if (elapsed >= 1) {
    remaining.value -= elapsed
    lastTick = now
  }
}

// 使用 setInterval(1000) 驱动，但每次 tick 用实际时间差校准
// 当页面从后台恢复时也能正确补偿
```

### 4.3 统计模块 (`useStats` / `StatsStore`)

#### 数据模型

```typescript
interface PomodoroRecord {
  id: string
  taskId?: string              // 关联的任务 ID
  startedAt: string            // ISO 日期时间
  endedAt: string              // ISO 日期时间
  duration: number             // 实际专注秒数

interface DailyStats {
  date: string                 // "YYYY-MM-DD"
  pomodoroCount: number        // 完成的番茄数
  totalMinutes: number         // 总专注分钟数
  completedTasks: number       // 完成的任务数

```

#### 统计维度

| 维度 | 说明 | 可视化 |
|------|------|--------|
| **今日概览** | 今日番茄数、专注时长、完成任务数 | 数字卡片 |
| **趋势分析** | 每日/每周专注时长变化 | 折线图（ECharts） |
| **打卡日历** | 连续专注天数热力图 | 日历热力图 |
| **标签分布** | 不同标签的番茄占比 | 饼图 |
| **任务效率** | 预估 vs 实际番茄数对比 | 柱状图 |

## 7. 关键设计决策

| 决策点 | 方案 | 理由 |
|--------|------|------|
| **计时精度** | `setInterval(1000)` + `Date.now()` 差值校准 | 避免 Tab 休眠或后台节流导致计时漂移 |
| **桌面框架** | Tauri v2（非 Electron） | 体积极小（~3MB vs ~150MB），Rust 后端性能好，系统资源占用低 |
| **数据存储** | IndexedDB (Web) / SQLite (Desktop) | 本地优先，无需后端服务器，离线可用 |
| **存储抽象** | StorageAdapter 接口 | Web / 桌面端存储引擎可切换，上层代码无感知 |
| **状态管理** | Pinia（非 Vuex） | Vue 3 官方推荐，原生 TS 支持，更轻量 |
| **图表库** | ECharts | 功能全面（折线图、热力图、饼图）、免费、社区活跃 |
| **音频方案** | Web Audio API + 预置 OGG 资源 | 无需网络请求，离线可用，支持多音源混合 |
| **计时器驱动** | setInterval 优先，requestAnimationFrame 备选 | setInterval 在后台 Tab 更可靠（配合时间校准） |
| **环形进度条** | SVG（非 Canvas） | 矢量清晰、CSS 可控、动画流畅 |
| **项目结构** | 按功能分层（views / components / composables / stores / storage） | 职责清晰，便于扩展和测试 |
